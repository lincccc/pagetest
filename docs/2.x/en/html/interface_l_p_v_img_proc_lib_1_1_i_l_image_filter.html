Detailed Description

This interface provide functionalities of various image filtering operations.

In a vision system, it's common to use image filtering operations to do pre-processing before detection and inspection task, in order to smooth, denoise, enhance or normalize the input image. The most common type of filters are linear, in which an output pixel's value is determined as a weighted sum of input pixel values:

\[ I'(x,y) = \sum_{i,j} f(x+i, y+j) k(i,j) \]

\( k(i,j) \) is called the filtering kernel.

To use this interface, you should create a LImageFilter object.

Example Code

C++
ILImageFilterPtr imgfilterOp = LImageFilter::Create();
imgfilterOp->SetKernelSize(5, 5);
imgfilterOp->Gaussian(img, resultImg);
C#
LImageFilter imgfilterOp = new LImageFilter();
imgfilterOp.SetKernelSize(5, 5);
imgfilterOp.Gaussian(img, resultImg);
COM
ILImageFilterPtr imgfilterOp = LImageFilter::Create();
// or, ILImageFilterPtr imgfilterOp(__uuidof(LImageFilter)), for absence of built-in Create functions (before 2.2.0)
imgfilterOp->SetKernelSize(5, 5);
imgfilterOp->Gaussian(img, resultImg);

◆ Gaussian()

void Gaussian ( ILImage *  img,
ILImage *  result 
)

Blurs an image using a Gaussian filter. Two x and y order Gaussian kernel is computed as a \( \texttt{ksize} \times 1 \) matrix of coefficients:

\[ G_i= \alpha *e^{-(i-( \texttt{ksize} -1)/2)^2/(2* \texttt{sigma}^2)}, \]

where \( i=0..\texttt{ksize}-1 \) , \( \alpha \) is the scale factor chosen so that \( \sum_i G_i=1 \) , \( sigma \) is set via SetKernelSigma() or computed from ksize as \( ksize \times 0.6 \) .

Parameters
[in]imgThe input image
[out]resultReturn result image

void LinearFilter ( ILImage *  img,

Apply linear filtering on the given image using the custom kernel kMat. The result pixel values out of depth range is truncated, for example for an 8bit image, negative result pixels are set to 0, greater than 255 are set to 255.

Parameters
[in]imgThe input image
[in]kWidthThe width of the custom kernel, in range 1 ~ 9999
[in]kHeightThe height of the custom kernel, in range 1 ~ 9999
[in]kMatData of the custom kernel, it's a collection of double values.
[out]resultReturn result image

Example Code

C++
// 1x5 kernel to extract vertical edges from black to white
imgFilter->LinearFilter(img, 5, 1, { -1, 0, 0, 0, 1 }, resultImg);
C#
// 1x5 kernel to extract vertical edges from black to white
imgFilter.LinearFilter(img, 5, 1, new double[] { -1, 0, 0, 0, 1 }, img2);
COM
// 1x5 kernel to extract vertical edges from black to white
CComSafeArray<double> kMat;
kMat.Add(-1); kMat.Add(0.); kMat.Add(0.); kMat.Add(0.); kMat.Add(1);
imgFilter->LinearFilter(img, 5, 1, kMat, resultImg);
Since
2.14.0

◆ SetKernelSigma()

void SetKernelSigma ( double  kSigma,
double  kGain 
)

Change the kernel using the specified sigma and gain value.

Parameters
[in]kSigmaThe sigma value is used to control the filter kernel's smoothness.
For HighPass(), Gaussian() and Sharpen() functions, if the sigma value is set to 0 then it's computed as \( 1/6 \) of the kernel size.
If the kernel size is set to all zeros, then we'll compute a proper size from the sigma value, it's around \( 6 \times kSigma \) .
For EdgePreserveDenoise() and EdgePreserveTextureEnhance(), set sigma value to 0 means no extra smoothing.
[in]kGainThe gain value is used to for Sharpen() to controls the overall strength of the sharpening effect.
See also
SetKernelSize(), HighPass(), Gaussian(), Sharpen()

Change kernel size used in the filter, by default it's a \( 3 \times 3 \) kernel. Large kernel size results usually in stronger filtering effect, but also slower.
You may set the kernel size to 0 then we'll compute a proper size from the sigma value.
