详细描述

该接口提供一组斑点分析结果的集合，包含一个或多个斑点。

通常通过调用 ILBlobAnalysis::Build() 得到该检测结果集合。 可以通过该接口访问其中的单个检测结果，绘制检测结果到给定的设备。

示例代码

C++

Code snippets: snippets/BlobOperation.cpp

ILBlobResultsPtr blobResults;
LPVErrorCode err = blobAna->Build(img, roi, &blobResults);
if (err != LPVErrorCode::LPVNoError) return;
int c = blobResults->Count();
for (int i = 0; i < c; ++i) {
ILBlobPtr blob = blobResults->Item(i);
// handle each blob result
}

◆ Intersect()

ILBlobResults* Intersect ( ILBlobResults *  other)

将当前斑点集合与输入的斑点集合进行并操作，生成一个新的斑点集合。
新的斑点仅包含两个集合同时覆盖的部分。一个原斑点若有多处相交，这些相交的部分在结果中仍旧属于同一个斑点。 若一个斑点不再包含任何有效像素，将被从集合中移除。因此，集合中斑点的数量可能会改变。

参数
[in]other输入用于并操作的斑点集合
返回值
newBlobResults输出一个新的斑点集合
参见
IntersectTo(), Subtract(), Union(), XOR().

◆ Union()

ILBlobResults* Union ( ILBlobResults *  other)

将当前与输入的斑点集合进行与操作（合并），生成一个新的斑点集合。
新集合中的斑点包含当前和输入斑点集合中的所有部分。重叠的斑点将被合并为一个斑点，独立的斑点将被保留。因此，集合中斑点的数量可能会改变。

◆ XOR()

ILBlobResults* XOR ( ILBlobResults *  other)

将当前斑点集合与输入的斑点集合进行异或操作，生成一个新的斑点集合。
新的斑点仅包含属于当前斑点但不属于输入斑点或属于输入斑点但不属于当前斑点的部分。 为了避免产生重复的斑点区域，other 中的每个斑点仅允许与一个当前集合中的斑点进行异或操作。
