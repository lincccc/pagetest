常见问题


编译错误

Q1. C++ 错误 C3506：没有为 LIBID XXX 注册的类型库

该错误表示对应平台的 LPV 模块在该机器上并未成功注册。

  1. 首先，确认 LPV 所在的文件夹路径**不包含任何特殊字符**，如 @ ( ) * & 。命名该路径中包含特殊字符的文件夹。
  2. 重新注册 LPV 模块，以管理员运行 runner/reg_run_as_admin.bat 脚本。

Q2. C++ error MSB8036: 找不到 Windows SDK 版本 XX.XX

如果 Visual Studio 或 MSBuild 在预期位置找不到项目请求的 Windows SDK，会发生此错误。
演示程序工程的目标平台为 10.0.16299.0 版本。若您并未安装对应的 Windows SDK，可修改工程对应的目标平台。右键工程打开工程配置界面，在 配置属性 > 常规 > 目标平台 ，修改为已安装的版本。

Q3. C++ error MSB8020: 无法找到 Visual Studio XXX 的生成工具(平台工具集 =“vXXX”)

若要解决此问题，请将项目升级为使用当前 Visual Studio 工具集，或使用 Visual Studio 安装程序安装指定的生成工具集。
演示程序工程使用 Visual Studio 2017 (v141) 工具集。如您并未安装 VS 2017 或者 2015，可修改工程对应的平台工具集。右键工程打开工程配置界面，在 配置属性 > 常规 > 平台工具集 ，修改为已安装的版本。

Q4. C# error CS0246: 找不到类型或命名空间名称 'LPVXXXLib' （是否缺少 using 指令或程序集引用?）

与 Q1 相同，其原因为对应平台的 LPV 模块在该机器上并未成功注册。请按 Q1 步骤排查和解决问题。


运行错误

Q5. C++ 调用 CreateDispatch 失败，返回错误 0x800401F0 尚未调用 CoInitialize

需要在程序开始处调用 CoInitializeEx，结束处调用 CoUnitialize。 需要在使用 LPV 模块前调用初始化。以 MFC 程序为例，推荐的调用位置为 MFC 程序的 InitInstance 和 ExitInstance 方法中。

例如:

BOOL CMFCApp::InitInstance()
{
CoInitializeEx(NULL, COINIT_MULTITHREADED);
// etc.
CWinApp::InitInstance();
}
INT CMFCApp::ExitInstance()
{
CoUninitialize();
return CWinApp::ExitInstance();
}

Q6. C++ 调用 CreateDispatch 失败，返回 False 且 m_lpDispatch 为空指针

与 Q1 相同，其原因为对应平台的 LPV 模块在该机器上并未成功注册。请按 Q1 步骤排查和解决问题。

Q7. C# 及其他 .net 调用 LPV 显示控件，编辑器报错：找不到 'AxLPVDisplayLib.AxLDisplay'，请确保已引用包含此类型的程序集。

对 x86 平台，只需重新编译该程序的 x86(32bit) 平台，然后重启 Visual Studio。

对 x64 平台，因为 Visual Studio 本身是 32 位的，其不能使用 x64 版本的控件（如 LDisplay 控件）进行图形化编辑。然后按照如下步骤：

  1. 重新注册 LPV 模块的 32 位版本，以管理员运行 runner/reg_run_as_admin.bat 脚本。
  2. 修改解决方案平台为 x86(32bit) 或者 AnyCPU
  3. 删除解决方案文件下的 obj/bin 文件夹。
  4. 清空和重新编译该解决方案。
  5. 重启 Visual Studio
  6. 修改解决方案平台回到 x64(64bit)
  7. 打开编辑器，确认错误是否已消除
