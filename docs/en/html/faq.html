FAQ


Compile Failure

Q1. C++ error C3506: there is no typelib registered for LIBID XXX

The error indicates the LPV libraries are not well registered for your platform.

  1. Make sure there's NO special character in the path of the registration file, like @ ( ) * & . Rename the folder with special characters.
  2. Re-register by run the runner/reg_run_as_admin.bat script as administrator.

Q2. C++ error MSB8036: The Windows SDK version XX.XX was not found.

The error occurs when Visual Studio or MSBuild can't find the Windows SDK requested by your project in the expected location.
The demo projects target the 10.0.16299.0 Windows SDK. If you don't have that version installed, you may right-click on the project node, under the Configuration Properties > General > Target Platform, select any available one.

Q3. C++ error MSB8020: The build tools for Visual Studio XXXX (Platform Toolset = 'vXXX') cannot be found.

To resolve this issue, upgrade the project to use the current Visual Studio toolset, or use the Visual Studio Installer to install the specified build toolset.
The demo projects use Visual Studio 2017 (v141) toolset. If you don't have VS 2017 or 2015 installed, you may right-click on the project node, under the Configuration Properties > General > Platform Toolset, change to any available one.

Q4. C# error CS0246: The type or namespace name 'LPVXXXLib' could not be found (are you missing a using directive or an assembly reference?)

Same as Q1, it's due to the LPV libraries are not well registered, please follow the steps in Q1.

Execution Failure

Q5. Call CreateDispatch fail with error 800401F0 CoInitialize has not been called. (C++ only)

You need to call CoInitializeEx when your program starts and CoUnitialize when it terminates. You should do it before create and initialize LPV classes. The best place to do this is in InitInstance and ExitInstance of your MFC application.

For example:

BOOL CMFCApp::InitInstance()
{
CoInitializeEx(NULL, COINIT_MULTITHREADED);
// etc.
CWinApp::InitInstance();
}
INT CMFCApp::ExitInstance()
{
CoUninitialize();
return CWinApp::ExitInstance();
}

Q6. Call CreateDispatch fail and return 0 with NULL m_lpDispatch. (C++ only)

Same as Q1, it's due to the LPV libraries are not well registered, please follow the steps in Q1.

Q7. Open the .net form with LPV display control, the designer report error: Could not find type 'AxLPVDisplayLib.AxLDisplay'.

If you are using the x86 LPV libraries. You just need to compile the project in x86(32bit), then restart Visual Studio.

If you are using the x64 LPV libraries. Since Visual Studio is natively 32bit, it cannot do GUI editing in some components (e.g. LDisplay control) in 64bit. To resolve this error, you should:

  1. Register the 32bit LPV libraries if you have not. Run runner/reg_run_as_admin.bat script as administrator.
  2. Change the solution to x86(32bit) or AnyCPU.
  3. Delete the obj/bin folder of the solution.
  4. Clean and rebuild the solution.
  5. Restart Visual Studio.
  6. Change the solution back to x64(64bit).
  7. Open the designer for GUI editing.
