Detailed Description

This interface provide functionalities of threshold operations.

Thresholding is a classic segmentation method to separate out regions of an image corresponding to objects which we want to analyze. This separation is based on the variation of intensity between the object and the background pixels. To differentiate the pixels, we perform a comparison of each pixel intensity value with respect to a threshold value (which is user-specified or determined by adaptive method)

To use this interface, you should create a LImageThreshold object.

Example Code

C++
ILImageThresholdPtr m_threshold_op(__uuidof(LImageThreshold));
m_threshold_op->SetThreshold(100, 200);
m_threshold_op->Binarize(m_img, m_result_img);
C#
LImageThreshold m_threshold_op = new LImageThreshold();
m_threshold_op.SetThreshold(100, 200);
m_threshold_op.Binarize(m_img, m_result_img);

Binarize the input image and return the result new image.
If the lower bound of threshold value is smaller than the upper bound, pixels within the lower and upper bound of threshold value is set to the object level (255), otherwise, the output pixel is set to the background level (0).
If the lower bound of threshold value is greater than the upper bound, the output result is inverted.

Use global-adaptive threshold value.
The optimal threshold value is calculated base on the entire input image, then revised by the given bias value. The lower bound of threshold value is \(t - lbBias\), the upper bound is \(t + ubBias\).

Parameters
lbBiasRevise value to optimal threshold value, to generate a threshold lower bound.
ubBiasRevise value to optimal threshold value, to generate a threshold upper bound.

int  blockHeight,

Use local-adaptive threshold value.
The optimal threshold value is calculated base on a \(blockWidth \times blockHeight\) region around individual pixel, then revised by the given bias value. The lower bound of threshold value is \(t - lbBias\), the upper bound is \(t + ubBias\).

Parameters
blockWidthThe width of a pixel's local region, should be an odd number, say, 3, 5, 7, etc.
blockHeightThe height of a pixel's local region, should be an odd number, say, 3, 5, 7, etc.
lbBiasRevise value to optimal threshold value, to generate a threshold lower bound.
ubBiasRevise value to optimal threshold value, to generate a threshold upper bound.

Stretch the pixel value within the lower and upper bound of threshold value, others is assigned to 0 or 255 respectively.
For example, if the threshold lower bound is 100, upper bound is 200, pixel is assigned to 0 if its input value is within range [0, 100], it's assigned to 255 if within range [200, 255], it's assigned \(p \times 255 / 200\) if within range [101, 199].
